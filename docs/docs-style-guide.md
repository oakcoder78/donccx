# Documentation Style Guide — doncCX Hub

## Purpose

Define the standard structure used across all documentation.

All generated documentation must follow this format.

---

# Module Documentation Standard

Target location:

docs/modules/<module-name>.md

Structure:

# <Module Name>

## Purpose

Describe:

- What this module does
- Why it exists

---

## Responsibilities

Describe:

- Core responsibilities
- Functional scope

---

## Key Components

List:

- Main components
- Hooks
- Services used

---

## Data Interaction

Describe:

- Tables accessed
- API usage
- External integrations

---

## UI Behavior

Describe:

- User-facing behavior
- Navigation flow

---

## Dependencies

List:

- Related modules
- Shared utilities

---

# System Documentation Standard

Target location:

docs/system/

Structure:

# <System Topic>

## Overview

High-level description.

---

## Architecture Role

Describe system position.

---

## Integration Points

List connected modules/services.

---

## Data Flow

Describe input/output flow.

---

# Activity / Step Documentation Standard

Target location:

docs/activity-attachments/

Structure:

# Step <Number> — <Name>

## Objective

What this step accomplishes.

---

## Implementation

Technical description.

---

## Validation

How to verify correctness.
