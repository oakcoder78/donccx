# Greeting Engine Roadmap v2.0

> Strategic evolution roadmap for the Greeting Engine subsystem.
>
> This document supersedes the original roadmap as the primary strategic direction of the project.
>
> The original `greeting-engine-roadmap.md` should be preserved as a historical roadmap documenting the foundational vision and early architectural assumptions.

---

# Vision

The Greeting Engine is no longer considered a simple greeting feature.

It evolved into:

```text
contextual narrative infrastructure
```

Its purpose is not:
- motivation
- engagement gimmicks
- chatbot interaction
- conversational AI

Its purpose is:

```text
create silent operational atmosphere through contextual narrative orchestration
```

The system should:
- enrich the dashboard experience
- humanize the interface subtly
- reinforce operational rhythm
- provide contextual atmosphere
- remain restrained and professional

The system should NEVER:
- dominate attention
- behave like a copiloto
- simulate emotional intimacy
- create fake urgency
- sound motivational
- compete with operational metrics

---

# Strategic Direction

The evolution of the Greeting Engine should prioritize:

```text
maturity over expansion
```

The system already proved:
- architectural extensibility
- deterministic orchestration
- contextual composition
- modular runtime design
- editorial governance viability

The next challenge is no longer technical feasibility.

The next challenge is:

```text
contextual maturity
```

---

# Core Principles

## 1. Silence is valuable

Not every context requires narrative output.

The system should feel restrained.

---

## 2. Atmosphere over messaging

The Greeting Engine exists to shape perception subtly.

It should not feel like a notification system.

---

## 3. Context without intrusion

Contextual awareness must remain:
- calm
- indirect
- non-invasive

---

## 4. Narrative consistency matters more than feature count

A smaller coherent system is preferable to a larger inconsistent system.

---

## 5. Narrative systems mature slowly

Editorial maturity requires:
- observation
- repetition analysis
- semantic refinement
- real-world usage

---

## Goal

Stabilize the narrative identity of the system through real-world usage.

---

## Focus Areas

- repetition analysis
- semantic fatigue
- contextual awkwardness
- subtlety refinement
- rhythm consistency
- operational atmosphere

---

## Deliverables

- narrative observation logs
- editorial refinements
- phrase pool cleanup
- contextual balance adjustments

---

## Success Criteria

The Greeting Engine should:
- feel natural
- avoid fatigue
- remain calm
- avoid noticeable repetition
- integrate invisibly into workflow

# Phase C — Content Externalization

## Goal

Decouple narrative content from runtime implementation.

---

## Focus Areas

- external content storage
- content versioning
- runtime loading
- environment-based content management
- editorial/runtime separation

---

## Deliverables

- content registry
- externalized pools
- runtime-safe loaders
- validation tooling

---

## Constraints

Externalization must NOT:
- introduce runtime instability
- compromise determinism
- create editorial chaos

---

# Phase D — Contextual Expansion

## Goal

Expand contextual awareness carefully.

---

## Potential Contexts

- onboarding stage
- lifecycle stage
- renewal proximity
- account maturity
- portfolio evolution
- activity rhythm

---

## Constraints

New layers must:
- remain subtle
- avoid over-personalization
- avoid behavioral creepiness
- preserve calm operational tone

---

# Phase E — Environmental Awareness

## Goal

Introduce environmental context carefully and minimally.

---

## Potential Inputs

- weather
- seasonal changes
- regional holidays
- workday patterns
- time context refinement

---

## Critical Constraint

Environmental awareness must NEVER:
- feel gimmicky
- feel assistant-like
- feel “smart for the sake of smart”

Examples to avoid:

```text
"Parece que vai chover hoje :)"
```

```text
"Hoje está frio em Florianópolis!"
```

These break the intended product sophistication.

---

# Phase F — Narrative Intelligence

## Goal

Introduce controlled narrative intelligence without transforming the system into a copiloto.

---

## Potential Capabilities

- contextual summarization
- semantic synthesis
- narrative prioritization
- adaptive orchestration

---

## Constraints

The system must NEVER:
- become conversational
- become agentic
- generate operational recommendations
- simulate personality
- behave like chat-based AI

---

# Phase G — Autonomous Context Systems

> **Nota (2026-05-19):** Phase G foi removida do roadmap ativo por contradizer o princípio "MUST NEVER become agentic / behave like a copiloto". Orquestração autônoma de múltiplos sinais é comportamento de copiloto disfarçado. Se houver demanda futura, revisitar com design explícito que não viole os princípios do sistema. Ver SDD (`greeting_engine_sdd.md`).

## Goal

Allow the Greeting Engine to orchestrate multiple weak contextual signals simultaneously.

---

## Examples

- temporal + operational
- lifecycle + seasonal
- onboarding + engagement rhythm
- renewal + portfolio movement

---

## Long-Term Vision

The Greeting Engine should eventually feel like:

```text
ambient operational intelligence
```

and NOT:

```text
an assistant speaking to the user
```

# Explicitly Deferred

The following remain intentionally deferred:

- emotional personalization
- mood detection
- persuasive narratives
- engagement optimization
- productivity coaching
- motivational language
- AI-generated hype
- autonomous recommendations
- behavioral manipulation

These directions conflict with the subsystem philosophy.

# Current Architectural Position

The Greeting Engine is currently considered:

```text
architecturally mature
but narratively maturing
```

This means:
- architecture is no longer the primary bottleneck
- editorial quality is now the primary concern
- contextual subtlety is now the main challenge

---

# Current Recommendation

The current recommendation is:

```text
observe before expanding
```

Focus should remain on:
- real-world usage
- repetition analysis
- narrative fatigue
- semantic consistency
- contextual elegance

before introducing:
- new layers
- environmental awareness
- narrative intelligence
