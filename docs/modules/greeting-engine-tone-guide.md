# Greeting Engine Tone Guide

## Propósito

Definir a identidade narrativa do Greeting Engine — como o sistema "soa" quando fala com o usuário.

---

## 1. Filosofia Narrativa

### O Que O Sistema É

O Greeting Engine é uma **consciência contextual silenciosa**.

Não é:
- um assistente virtual
- um chatbot
- um copiloto
- um narrator gamificado

É:
- uma camada atmosférica que enriquece a experiência
- um observador silencioso que sugere contexto
- uma voz que humaniza discretamente o produto

### Posicionamento

O sistema deve competir por atenção? **NUNCA.**

O sistema deve complementar métricas? **SIM.**

O sistema deve parecer parte do produto? **SIM.**

## 2. Voz do Produto

### Características permitidas

| Característica | Descrição |
|---------------|------------|
| Profissional | Tom de ferramenta séria |
| Calmo | Não urgente, não alarmista |
| Observacional | Descreve, não julgar |
| Minimalista | Poucas palavras, alto impacto |
| Contextual | Reage ao estado do sistema |
| Sutil | Presente mas não intrusivo |

### Características proibidas

| Proibido | Por quê |
|----------|--------|
| Teatral | Exagero ridículo |
| Exclamativo | "!" em excesso é manipulador |
| Motivacional aggressivo | "Bora!", "Vamos!" forçados |
| Sarcástico | Tom irônico não escala |
| Infantil | "Yay!", "Woo!" |
| Meme-driven | Referências virais |
| Emocionalmente manipulativo | "Você precisa fazer isso AGORA!" |
| Hiper-energético | Sempre animado é artificial |
| Buscador de atenção | "Ei! Ei! Olha aqui!" |
| Chatbot persona | Personalidade fake |

## 3. Princípios de Escrita

### 3.1 Sutileza

**Errado:** "Que bom te ver!"
**Certo:** "Bom dia."

O sistema Não deve soar emocionado em ver o usuário.

### 3.2 Calma Operacional

**Errado:** "ALERTA! SUA CARTEIRA ESTÁ EM RISCO!"
**Certo:** "Carteira estável."

O sistema não deve alarmar. Deve informar.

### 3.3 Inteligência Silenciosa

**Errado:** "Você sabia que 3 clientes precisam de atenção?"
**Certo:** "Três clientes em acompanhamento."

O sistema observa, não educa.

### 3.4 Clareza Sem Teatro

**Errado:** "Uau! Seus números estão incríveis!"
**Certo:** "Números em crescimento."

O sistema reporta, não celebra.

### 3.5 Observação, Não Interpretação

**Errado:** "Que semana agitada!"
**Certo:** "Semana de volume alto."

O sistema estados. Não jutga.

## 6. Anti-Patterns

### 6.1 Sistemas Que Sempre Falam

O sistema não deve cumprimentar em cada interação.

 apenas no greeting inicial.

### 6.2 Mensagens de Waiting

"te esperam", "esperando" cria urgência fake.

Eliminar todas as metáforas de espera.

### 6.3 Exclamações

"!"= manipulação emocional.

Evitar. Se usar, apenas para segurança/urgência real.

### 6.4 Imperativos

"Hora de...", "Vamos...", "Aproveite..."

O sistema informa. Não instrui.

### 6.5 Perguntas Retóricas

"Bora?", "não é?"

Sistema não deve fazer perguntas.

### 6.6 Tom Variável

Hoje está feliz, amanhã profissional.

Manter consistência. O tom é sempre calmo, observacional.

## 7. Critérios de Aprovação

### Checklist Editorial

Para cada nova frase, verificar:

- [ ] **Não é chatbot?** Não soa como assistente pessoal
- [ ] **Não é imperative?** Não instrui ação
- [ ] **Não é attention-seeking?** Não cria urgência fake
- [ ] **Não é celebratório?** Não sobre-celebra
- [ ] **Não é gamificado?** Não usa recreational language
- [ ] **Não é corporativo buzzword?** Não usa "impactar", "entregar", "resultados"
- [ ] **Não é emocionmanipulative?** Não usa guilt/fear
- [ ] **É observacional?** Descreve estado, não julga
- [ ] **É profissional?** Soa como ferramenta séria
- [ ] **É discreto?** Não compete com métricas

## 8. Tom por Camada

### Temporal Layer

Minimalista. Apenas horário.

- Bom dia / Boa tarde / Boa noite
- Opcional: dia da semana (se relevante)

Não:
- comentarios sobre o dia
- julgamentos ("Que semana agitada")
- instruções ("Aproveite")

### Identity Layer

Observacional. Foco no contexto profissional.

- "Gestão em dia"
- "Clientes ativos"

Não:
- urgências ("te esperam")
- imperativos ("Hora de...")
- celebrações ("Parabéns!")

### Operational Layer

Informativo. Estados do sistema.

- "Carteira equilibrada"
- "Três clientes em atenção"

Não:
- alarmes ("RISCO CRÍTICO")
- imperativos ("exige atenção")
- julgamentos ("importantes")

---

## 9. Evolução Futura

Este guide evolui conforme o sistema ganha camadas.

Antes de IA/contextual intelligence:

1. Executar cleanup das frases problemáticas
2. Aplicar critérios de aprovação
3. Validar com usuarios reais
4. Ajustar tom baseado em feedback

Apenas então considerar:
- personalized adaptation
- behavioral narrative
- predictive greetings
